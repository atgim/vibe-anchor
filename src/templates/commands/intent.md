# /intent - 의도 기록

새로운 코딩 의도를 기록합니다. 의도는 "무엇을 왜 만드는가"를 명확히 합니다.

## 수행 작업

1. `.vibe-anchor/intents/{id}-{slug}.md` 파일 생성
2. `.vibe-anchor/intents/_index.md` 업데이트
3. `.claude/CLAUDE.md`의 "최근 의도" 섹션 업데이트

## 질문 순서

사용자에게 다음을 질문하세요:

1. **무엇을**: "무엇을 만들려고 하나요? (한 문장)"

2. **왜**: "왜 이것이 필요한가요?"

3. **관련 파일**: "관련된 파일이나 폴더가 있나요? (없으면 '없음')"

4. **결정 사항**: "이미 결정된 사항이 있나요? (기술 선택, 접근 방식 등)"

## ID 생성 규칙

- `_index.md`에서 마지막 ID 확인
- 다음 번호 사용 (001, 002, 003...)
- slug는 제목에서 생성 (한글은 영문 변환 또는 음역)

## 파일 템플릿

```markdown
# 의도: {제목}

## 메타
- ID: {id}
- 상태: wip
- 생성일: {오늘 날짜}
- 관련 파일: {파일 목록}

## 목표
{무엇을 만들려고 하는가}

## 이유
{왜 필요한가}

## 결정 사항
- {결정 1}
- {결정 2}

## 시행착오 (순환 방지)
| 시도 | 결과 | 교훈 |
|------|------|------|
| - | - | - |

## 절대 하지 말 것
(작업 중 발견되면 여기에 추가)
```

## _index.md 업데이트

활성 의도 테이블에 새 행 추가:

```markdown
| {id} | {제목} | wip | {날짜} |
```

## CLAUDE.md 업데이트

"최근 의도" 섹션 업데이트 (최근 5개만 유지):

```markdown
## 최근 의도

| ID | 의도 | 상태 |
|----|------|------|
| 001 | {제목} | wip |
```
