# /review - 세션 정리

현재 세션을 정리하고 컨텍스트를 저장합니다.

## 수행 작업

1. 세션 요약 생성
2. `.vibe-anchor/sessions/{날짜}.md` 생성/업데이트
3. `.claude/CLAUDE.md` 업데이트
4. 의도/태스크 상태 정리

## 세션 요약 수집

다음 정보를 수집합니다:

### 1. 오늘 완료한 작업
- 완료된 태스크 목록
- 완료된 TDD 사이클 수
- 생성/수정된 파일 목록

### 2. 시행착오
- 실패한 접근법
- 얻은 교훈
- 앞으로 피해야 할 것

### 3. 다음 세션을 위한 TODO
- 미완료 태스크
- 다음에 할 일
- 주의사항

### 4. 메모
- 사용자에게 질문: "오늘 작업하면서 기억해둘 것이 있나요?"

## 세션 파일 생성

`.vibe-anchor/sessions/{YYYY-MM-DD}.md`:

```markdown
# 세션: {날짜}

## 요약

- 완료 태스크: {n}개
- TDD 사이클: {n}회
- 커밋: {n}개

## 완료한 작업

- [x] {태스크 1}
- [x] {태스크 2}

## 시행착오

| 시도 | 결과 | 교훈 |
|------|------|------|
| {시도} | 실패 | {교훈} |

## 다음 세션 TODO

- [ ] {할 일 1}
- [ ] {할 일 2}

## 변경된 파일

- `src/...` (생성)
- `src/...` (수정)

## 메모

{사용자 메모}
```

## CLAUDE.md 업데이트

### 세션 노트 섹션

최근 5개 세션만 유지:

```markdown
## 세션 노트

- 2026-01-18: 로그인 기능 구현 시작, 폼 컴포넌트 완료
- 2026-01-17: 프로젝트 초기화
```

### 현재 작업 섹션

미완료 태스크가 있으면:
```markdown
## 현재 작업

- 태스크: T002 - 이메일 유효성 검사
- 상태: 대기 (이전 세션에서 중단)
- 의도: 001 - 로그인 기능
```

### 수정 금지 사항 확인

새로 발견된 제약이 있으면 추가 여부 확인:
```
세션 중 발견된 수정 주의 사항:
- src/auth/token.ts는 보안 관련 코드입니다.

이것을 수정 금지 사항에 추가할까요?
```

## 의도 상태 업데이트

완료된 의도가 있으면:
1. `intents/{id}.md`의 상태를 `done`으로 변경
2. `intents/_index.md` 업데이트 (활성 → 완료)

## 체크리스트 출력

세션 종료 전 확인:

```
📋 세션 종료 체크리스트

- [ ] 모든 변경사항 커밋됨
- [ ] 테스트 통과 확인
- [ ] 시행착오 기록됨
- [ ] 다음 TODO 명확함

세션을 종료할까요?
```

- [ ] 의도/태스크 상태 정리
- [ ] 커밋 여부 확인
