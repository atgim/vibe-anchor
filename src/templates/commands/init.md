# /init - 프로젝트 초기화

프로젝트의 핵심 원칙과 기술 스택을 정의합니다.

## 수행 작업

`.claude/CLAUDE.md`의 "Constitution" 섹션을 작성합니다.

## 질문 순서

사용자에게 다음을 순서대로 질문하세요:

### 1. 프로젝트 목표
"이 프로젝트의 목표를 한 문장으로 설명해주세요."

### 2. 기술 스택
"사용할 기술 스택을 알려주세요."
- 언어: (예: TypeScript, Python)
- 프레임워크: (예: React, FastAPI)
- 데이터베이스: (예: PostgreSQL, SQLite)

### 3. 핵심 원칙
"이 프로젝트에서 지켜야 할 핵심 원칙 2-3가지를 알려주세요."
예시:
- TDD 필수
- 타입 안전성 우선
- 라이브러리 최소화

### 4. 코딩 스타일
"선호하는 코딩 스타일이 있나요?"
- 네이밍 규칙
- 파일 구조
- 포매터/린터

## 파일 업데이트

질문 응답을 바탕으로 `.claude/CLAUDE.md`의 "Constitution" 섹션 업데이트:

```markdown
## Constitution (프로젝트 원칙)

### 프로젝트 목표
{목표}

### 기술 스택
- **언어**: {언어}
- **프레임워크**: {프레임워크}
- **데이터베이스**: {DB}

### 핵심 원칙
1. **{원칙1 이름}**: {설명}
2. **{원칙2 이름}**: {설명}
3. **{원칙3 이름}**: {설명}

### 코딩 스타일
- 네이밍: {규칙}
- 파일 구조: {구조}
- 포매터: {도구}
```

## 예시 출력

```markdown
## Constitution (프로젝트 원칙)

### 프로젝트 목표
사용자가 할 일을 관리할 수 있는 간단한 투두 앱

### 기술 스택
- **언어**: TypeScript
- **프레임워크**: React + Vite
- **데이터베이스**: localStorage (MVP)

### 핵심 원칙
1. **TDD 필수**: 모든 기능은 테스트를 먼저 작성한 후 구현
2. **타입 안전성**: strict 모드, any 타입 금지
3. **단순함 우선**: 복잡한 추상화보다 명확한 코드

### 코딩 스타일
- 네이밍: camelCase (변수/함수), PascalCase (컴포넌트)
- 파일 구조: feature 기반 (components/Login/, hooks/useAuth)
- 포매터: Prettier + ESLint
```
