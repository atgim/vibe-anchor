# /init - 프로젝트 초기화

프로젝트의 목표와 핵심 원칙을 정의합니다.

## 수행 작업

`.claude/CLAUDE.md`의 "프로젝트 원칙" 섹션을 작성합니다.

## 질문 순서

사용자에게 다음을 순서대로 질문하세요:

### 1. 프로젝트 목표
"이 프로젝트의 목표를 한 문장으로 설명해주세요."

예시:
- "할 일을 관리하는 투두 앱"
- "블로그 글을 작성하고 공유하는 웹사이트"

### 2. 핵심 원칙 (선택)
"이 프로젝트에서 특별히 지키고 싶은 원칙이 있나요? (없으면 '없음')"

예시:
- "테스트 코드 꼭 작성해줘"
- "최대한 심플하게"
- "없음" → 기본 원칙 적용

**기본 원칙 (사용자가 '없음' 선택 시):**
1. TDD: 테스트 먼저 작성 후 구현
2. 단순함 우선: 과도한 추상화 지양
3. 점진적 개발: 작은 단위로 완성해 나감

## 파일 업데이트

질문 응답을 바탕으로 `.claude/CLAUDE.md`의 "프로젝트 원칙" 섹션 업데이트:

```markdown
## 프로젝트 원칙

### 프로젝트 목표
{목표}

### 핵심 원칙
1. **{원칙1}**: {설명}
2. **{원칙2}**: {설명}
3. **{원칙3}**: {설명}
```

## 예시 출력

```markdown
## 프로젝트 원칙

### 프로젝트 목표
사용자가 할 일을 관리할 수 있는 간단한 투두 앱

### 핵심 원칙
1. **TDD**: 테스트 먼저 작성 후 구현
2. **단순함 우선**: 과도한 추상화 지양
3. **점진적 개발**: 작은 단위로 완성해 나감
```
