# /init - 프로젝트 초기화

프로젝트의 핵심 원칙과 기술 스택을 정의합니다.

## 수행 작업

`.vibe-anchor/constitution.md` 파일을 작성합니다.

## 질문 순서

사용자에게 다음을 순서대로 질문하세요:

1. **프로젝트 목표**: "이 프로젝트의 목표를 한 문장으로 설명해주세요."

2. **기술 스택**: "사용할 기술 스택을 알려주세요. (언어, 프레임워크, DB 등)"

3. **핵심 원칙**: "이 프로젝트에서 지켜야 할 핵심 원칙 3가지를 알려주세요. (예: TDD 필수, 타입 안전성 우선 등)"

4. **코딩 스타일**: "선호하는 코딩 스타일이 있나요? (네이밍 규칙, 파일 구조 등)"

## 파일 업데이트

질문 응답을 바탕으로 다음 파일들을 업데이트하세요:

1. `.vibe-anchor/constitution.md` - 전체 내용 작성
2. `.claude/CLAUDE.md` - "프로젝트 원칙" 섹션 업데이트

## 예시 출력

```markdown
# 프로젝트 헌법: my-app

## 핵심 원칙

### 1. TDD 필수
모든 기능은 테스트를 먼저 작성한 후 구현합니다.

### 2. 타입 안전성
TypeScript strict 모드를 사용하고, any 타입 사용을 금지합니다.

### 3. 단순함 우선
복잡한 추상화보다 명확하고 단순한 코드를 선호합니다.

## 기술 스택
- 언어: TypeScript
- 프레임워크: React
- 테스트: Vitest
- 빌드: Vite

## 프로젝트 목표
사용자가 할 일을 관리할 수 있는 간단한 투두 앱
```
