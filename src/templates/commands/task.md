# /task - 태스크 관리

작은 단위의 태스크를 정의하고 관리합니다.

## 사용법

- `/task` - 새 태스크 생성
- `/task list` - 백로그 확인
- `/task done` - 현재 태스크 완료

## 새 태스크 생성

### 질문 순서

1. **태스크**: "무엇을 할 건가요? (한 문장)"

2. **완료 조건**: "언제 완료인가요? (구체적으로)"

3. **관련 의도**: "관련된 의도 ID가 있나요? (없으면 '없음')"

### 태스크 원칙 체크

태스크 생성 전 사용자에게 확인:

> "이 태스크가 다음 원칙에 맞는지 확인해주세요:
> - [ ] 짧은 시간 내 완료 가능
> - [ ] 가능하면 파일 1-2개만 수정
> - [ ] 완료 조건이 명확함
>
> 너무 크면 분해해드릴까요?"

### 파일 업데이트

1. `.vibe-anchor/tasks/current.md` 업데이트:

```markdown
# 현재 태스크

## 진행 중인 태스크

**T{id}: {태스크 제목}**

## 태스크 정보

- ID: T{id}
- 제목: {태스크}
- 의도: {의도 ID 또는 없음}
- 시작일: {오늘}
- TDD 상태: 대기

## 완료 조건

- [ ] {조건 1}
- [ ] {조건 2}

## 메모

(작업 중 메모)
```

2. `.vibe-anchor/tasks/backlog.md`에서 해당 태스크 제거 (있었다면)

3. `.claude/CLAUDE.md`의 "현재 작업" 섹션 업데이트

## /task list

백로그를 보여줍니다:

```
📋 태스크 백로그

대기 중:
  T001: 로그인 폼 컴포넌트 생성 [의도: 001]
  T002: 이메일 유효성 검사 [의도: 001]

완료됨:
  (없음)
```

## /task done

현재 태스크를 완료 처리합니다:

1. `tasks/current.md` 초기화
2. `tasks/backlog.md`의 완료됨 섹션에 추가
3. `CLAUDE.md` 업데이트
4. 관련 의도의 진행 상황 업데이트

## 태스크 분해 도움

사용자가 큰 태스크를 제시하면 분해를 제안:

```
❌ "로그인 기능 구현" → 너무 큼

✅ 분해 제안:
  1. 로그인 폼 UI 컴포넌트 생성
  2. 이메일 입력 필드 유효성 검사
  3. 비밀번호 입력 필드 유효성 검사
  4. 로그인 API 호출 함수 작성
  5. 로그인 에러 처리 UI
  6. 로그인 성공 후 리다이렉트

어떤 것부터 시작할까요?
```
